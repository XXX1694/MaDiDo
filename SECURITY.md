# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in this project, please report it responsibly.

**Do NOT open a public GitHub issue for security vulnerabilities.**

Instead, please email: **[open an issue with the "security" label](https://github.com/XXX1694/MaDiDo/issues/new?labels=security)**

Or use GitHub's private vulnerability reporting:
1. Go to the [Security tab](https://github.com/XXX1694/MaDiDo/security)
2. Click "Report a vulnerability"
3. Fill in the details

## What to Include

- Description of the vulnerability
- Steps to reproduce
- Potential impact
- Suggested fix (if any)

## Response Timeline

- **Acknowledgment**: Within 48 hours
- **Initial assessment**: Within 1 week
- **Fix release**: Depends on severity, but we aim for prompt resolution

## Scope

This policy applies to the latest version of the application on the `main` branch.

## Supported Versions

| Version | Supported |
| ------- | --------- |
| Latest  | Yes       |
| Older   | No        |
